Сгенерируй структуру проекта и базовый код для Telegram-бота на aiogram 3.x. Вот полное техническое задание:"

Техническое задание: Telegram-бот "SENU Buddy"
1. Общее описание проекта
Цель: Создать Telegram-бота для упрощения коммуникации между студентами Назарбаев Университета (NU) и ментором организации SENU (Айнур). Бот должен снизить бюрократический барьер, автоматизировать запись на встречи, информировать о мероприятиях и маршрутизировать экстренные запросы.

Целевая аудитория: Студенты NU (бакалавриат и магистратура).
Администратор: Айнур (SENU Ментор).

2. Технический стек
Язык программирования: Python 3.10+

Фреймворк для Telegram: aiogram версии 3.x (строго 3.x, использовать асинхронность и роутеры).

База данных: SQLite (для MVP достаточно, легко перенести на PostgreSQL позже).

ORM: SQLAlchemy 2.0 (асинхронная сессия) или легковесный aiosqlite.

Управление состояниями: FSM (Finite State Machine) из aiogram.

Конфигурация: python-dotenv для чтения .env (токен бота, ID админа).

3. Архитектура базы данных (Таблицы)
Необходимо создать две основные таблицы:

Таблица users (Студенты):

telegram_id (Integer, Primary Key)

username (String, nullable)

full_name (String)

joined_at (DateTime)

Таблица requests (Заявки/Вопросы):

id (Integer, Primary Key, AutoIncrement)

user_id (Integer, Foreign Key к users)

request_type (String: 'meeting', 'game_108', 'question', 'anonymous_question')

content (Text: текст вопроса или желаемое время встречи)

status (String: 'pending', 'resolved', default='pending')

created_at (DateTime)

4. Ролевая модель
User (Студент): Имеет доступ к главному меню, может отправлять заявки и получать рассылки.

Admin (Ментор - Айнур): ID администратора указывается в .env (ADMIN_ID). Админ получает уведомления о новых заявках и может делать рассылку (broadcast) всем пользователям бота.

5. Пользовательский сценарий (Студент)
Команда /start

Сохраняет пользователя в БД (если его нет).

Отправляет приветственное сообщение: "Привет! Я бот-помощник Айнур, твоего SENU-ментора. Здесь ты можешь записаться на встречу, задать вопрос или найти нужные контакты."

Выводит постоянную клавиатуру (ReplyKeyboardMarkup) с кнопками.

Кнопки главного меню и их логика:

👋 О менторе

Выводит фотографию Айнур (ссылка на фото или file_id).

Текст: Краткая информация (Bolashak alumni, GWU, опыт в NU, фасилитатор игры «108»).

☕️ Записаться на встречу

Запускает FSM (состояния).

Бот: "Опиши вкратце, что бы ты хотел обсудить? (Академический вопрос, личный баланс и т.д.)" -> Ожидание текста.

Бот: "В какие дни и время тебе было бы удобно встретиться?" -> Ожидание текста.

Бот: "Заявка отправлена Айнур. Она свяжется с тобой в ближайшее время!"

Действие: Сохраняет в БД, отправляет уведомление Админу.

🎲 Игра «108»

Выводит описание трансформационной игры.

Показывает Inline-кнопку [Хочу сыграть].

При нажатии отправляет уведомление Админу: "Студент @username хочет сыграть в 108".

❓ Задать вопрос

Выводит Inline-клавиатуру: [Открыто (с моим юзернеймом)] и [Анонимно].

При выборе открытого: запускает FSM для ввода текста, отправляет Админу со ссылкой на профиль студента.

При выборе анонимного: запускает FSM, отправляет Админу без указания авторства.

🆘 Психологическая помощь (PCS)

Отправляет статический текст с контактами PCS.

Указывает Telegram бота @pcs_nu, телефоны доверия (111, 150) и напоминает, что психологический скрининг лучше проходить с ПК (вставляет ссылку на портал).

6. Административный сценарий (Айнур)
Уведомления о заявках
Когда студент заполняет форму встречи или задает вопрос, Админ (Айнур) получает сообщение вида:

🔔 Новая заявка: Встреча
От: Имя ( @username )
Тема: Проблемы с пробацией
Удобное время: Вторник после 15:00

К сообщению прикрепляется Inline-кнопка: [Отметить как решенное]. При нажатии статус в таблице requests меняется на 'resolved'.

Команда /broadcast (Рассылка)

Доступна только для ADMIN_ID.

Использование: Админ пишет /broadcast Всем привет, завтра в 16:00 зум по учебе за рубежом! Ссылка: ...

Действие: Бот рассылает этот текст всем telegram_id из таблицы users.

Бот отвечает админу: "Рассылка завершена. Доставлено: N пользователям."

7. Рекомендуемая структура проекта для Cursor
Создай в проекте следующие папки и файлы (можешь дать эту структуру Cursor, он сам создаст файлы):

bot.py (точка входа, запуск бота)

.env (токены и настройки)

requirements.txt (зависимости)

database/

models.py (модели SQLAlchemy)

db.py (инициализация сессий)

handlers/

user_handlers.py (обработка кнопок меню)

admin_handlers.py (обработка рассылки и уведомлений)

fsm_forms.py (машины состояний для записи на встречу/вопросов)

keyboards/

reply.py (главное меню)

inline.py (кнопки под сообщениями)




сначала написать models.py и bot.py, а затем пошагово реализовать хэндлеры.
